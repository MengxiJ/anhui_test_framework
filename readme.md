# 安汇智投金融平台 - 自动化测试框架

## 项目简介

基于PageObject设计模式的UI+API+性能一体化自动化测试框架，覆盖P2P金融平台核心业务流程，包括登录、注册、开户、充值、投资等功能。

## 快速开始

### 1. 环境配置

```bash
# 创建虚拟环境
python -m venv venv
venv\Scripts\activate  # Windows

# 安装依赖
pip install -r requirements.txt
```

### 2. 配置测试环境

编辑 `config.py`，配置测试环境地址：

```python
API_BASE_URL = "http://121.43.169.97:8081"
BASE_URL = "http://121.43.169.97:8081"
```

### 3. 运行测试

```bash
# 运行所有测试
python tools/run_tests.py --all

# 只运行API测试
python tools/run_tests.py --api

# 只运行UI测试
python tools/run_tests.py --ui

# 自动生成并打开Allure报告
python tools/run_tests.py --serve

# 并行运行测试
python tools/run_tests.py --all --parallel
```

### 4. 查看测试报告

```bash
# 自动生成报告并在浏览器中打开
python tools/cmd_allure.py

# 或使用run_tests.py
python tools/run_tests.py --serve
```

### 5. 运行性能测试

```bash
# 检查JMeter安装
python tools/run_performance.py --check

# 运行性能测试（自动生成报告）
python tools/run_performance.py --run

# 指定测试计划
python tools/run_performance.py --run --jmx performance/performance_test.jmx

# 从最新结果生成报告
python tools/run_performance.py --report
```

### 6. 生成测试统计报告

```bash
python tools/test_statistics.py
```

## API接口说明

### 1. 获取图片验证码

- **接口**: `GET /common/public/verifycode1/{r}`
- **参数**: r - 随机数
- **返回**: 图片

### 2. 获取短信验证码

- **接口**: `POST /member/public/sendSms`
- **参数**: phone, imgVerifyCode, type
- **返回**: {"status": 200/100, "description": "..."}

### 3. 注册

- **接口**: `POST /member/public/reg`
- **参数**: phone, password, verifycode, phone\_code, dy\_server, invite\_phone(可选)
- **返回**: {"status": 200/100, "description": "..."}

### 4. 登录

- **接口**: `POST /member/public/login`
- **参数**: keywords, password
- **返回**: {"status": 200/100, "description": "..."}

### 5. 判断是否登录

- **接口**: `POST /member/public/isLogin`
- **返回**: {"status": 200/250, "description": "..."}

## 性能测试文档

### 概述

本项目使用JMeter进行API性能测试，验证系统在高并发场景下的性能表现。

### 测试目标

- 验证系统在高并发下的响应时间
- 测试系统的吞吐量（TPS）
- 发现系统性能瓶颈
- 验证系统稳定性

### 安装 JMeter

#### Windows

1. 下载 JMeter: <https://jmeter.apache.org/download_jmeter.cgi>
2. 解压到指定目录
3. 配置环境变量：
   - `JMETER_HOME`: JMeter安装目录
   - `Path`: 添加 `%JMETER_HOME%\bin`

#### Mac/Linux

```bash
# 使用 Homebrew 安装
brew install jmeter

# 或手动下载安装
wget https://downloads.apache.org//jmeter/binaries/apache-jmeter-5.6.3.tgz
tar -xzf apache-jmeter-5.6.3.tgz
export PATH=$PATH:$(pwd)/apache-jmeter-5.6.3/bin
```

### 验证安装

```bash
jmeter --version
```

## CI/CD 自动化测试流水线

### 概述

本项目已集成完整的CI/CD自动化测试流水线，支持：

- GitHub Actions 自动化测试
- Jenkins 流水线配置
- 自动生成 Allure 测试报告
- 测试报告自动发布到 GitHub Pages
- 性能测试集成

### GitHub Actions 使用指南

#### 自动触发条件

1. **代码推送**: 推送到 `main`、`master`、`dev` 分支时自动触发
2. **Pull Request**: 创建 PR 到上述分支时自动触发
3. **定时任务**: 每天凌晨2点自动运行
4. **手动触发**: 在 GitHub Actions 页面手动运行

#### 查看测试报告

测试执行完成后，报告会自动发布到 GitHub Pages：

```
https://<your-username>.github.io/<repository-name>/
```

#### 工作流程

```
代码提交 -> 自动运行测试 -> 生成报告 -> 发布到GitHub Pages
```

### Jenkins 配置指南

#### 前置要求

1. **Jenkins 插件**:
   - Allure Jenkins Plugin
   - Pipeline
   - Git Plugin
2. **全局工具配置**:
   - Python 3.11
   - Allure Commandline

#### 创建 Jenkins 任务

1. 创建新的 Pipeline 任务
2. 配置 Git 仓库地址
3. 选择 "Pipeline script from SCM"
4. 指定 Jenkinsfile 路径

#### Jenkins 流水线阶段

```
代码检出 -> 环境准备 -> API测试 -> UI测试 -> 性能测试 -> 生成报告 -> 邮件通知
```
